# 019: オーバーレイ動画 MP4動画背景対応 — 要求仕様書

## 1. プロジェクト概要

- **何を作るのか**: `pose_overlay_video.py`の`-b`オプションをMP4動画ファイルにも対応させ、元動画のフレーム上にキーポイント・骨格を重ねた動画を生成できるようにする
- **なぜ作るのか**: 現状は静止画背景か黒背景のみ対応。元動画の上にキーポイントを重ねることで、推定精度の確認（人体の実際の位置とキーポイントのずれ）が直感的に行える
- **誰が使うのか**: Pose2Sim利用者（データ解析者）
- **どこで使うのか**: Linux環境、Pose2Sim Python環境内

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| 背景動画 | `-b`オプションで指定するMP4動画ファイル。各フレームがオーバーレイの背景になる |
| 背景画像 | `-b`オプションで指定する静止画ファイル（既存機能。全フレーム共通の背景） |
| フレーム対応 | 背景動画のフレームiとJSONファイルのi番目を1対1で対応させること |

## 3. 機能要求一覧

### FR-001: `-b`オプションの動画ファイル対応

- **概要**: `-b`オプションに画像ファイルだけでなくMP4動画ファイルも指定できるようにする。ファイルの拡張子で画像か動画かを判定する
- **入力**: ファイルパス（画像: `.jpg`, `.jpeg`, `.png`, `.bmp` / 動画: `.mp4`, `.avi`, `.mov`, `.mkv`）
- **出力**: 画像ファイルの場合は既存動作（静止画背景）、動画ファイルの場合はフレームごとの背景として使用
- **受け入れ基準**:
  - `pose_overlay_video -j /path/to/json_dir -b video.mp4` で動画背景が使用される
  - `pose_overlay_video -j /path/to/json_dir -b image.jpg` で既存の静止画背景が使用される（後方互換性）
  - `-b`未指定時は黒背景（既存動作）

### FR-002: フレーム単位の背景描画

- **概要**: 背景動画のi番目のフレームをi番目のJSONに対応させ、各フレームの動画画像上にキーポイント・骨格を描画する
- **入力**: 背景動画のフレーム、JSONのキーポイントデータ
- **出力**: 動画フレーム上にキーポイント・骨格が描画された画像
- **受け入れ基準**: 動画フレームとJSONが1対1で対応し、人体の位置にキーポイントが正しく重なっている

### FR-003: フレーム数の不一致対応

- **概要**: 背景動画のフレーム数とJSONファイル数が異なる場合の処理を定義する
- **入力**: 背景動画（N_vフレーム）、JSONファイル（N_jファイル）
- **出力**: min(N_v, N_j)フレームの出力動画
- **受け入れ基準**:
  - N_v > N_j の場合: JSONファイル数分（N_j）のフレームを出力し、残りの動画フレームは無視する。警告メッセージを出力する
  - N_v < N_j の場合: 動画フレーム数分（N_v）のフレームを出力し、残りのJSONファイルは無視する。警告メッセージを出力する
  - N_v = N_j の場合: 全フレームを出力する

### FR-004: 動画背景使用時の`--size`と`--fps`の扱い

- **概要**: 背景動画使用時は動画の解像度とFPSを自動取得する
- **入力**: 背景動画のメタデータ
- **出力**: 動画から取得した解像度とFPS
- **受け入れ基準**:
  - `--size`は無視される（動画の解像度を使用）。動画のリサイズ出力は本案件のスコープ外
  - `--fps`が未指定の場合は動画のFPSを使用する。`--fps`が明示的に指定された場合はその値を使用する

## 4. 非機能要求

- **パフォーマンス**: 動画背景使用時も10万フレームを10分以内に処理できる
- **後方互換性**: `-b`に画像ファイルを指定した場合、および`-b`未指定の場合は、現行動作と完全に同じ出力を生成する
- **対応環境**: Linux、Python 3.9+、GPU不要

## 5. 制約条件

- `Pose2Sim/Utilities/pose_overlay_video.py`内の変更のみで完結させる
- 動画読み込みは`cv2.VideoCapture`を使用する（Pose2Sim既存依存のOpenCV）
- 新規ライブラリの追加は不要

## 6. 優先順位

| ID | 優先度 | 備考 |
|----|--------|------|
| FR-001 | Must | 動画ファイルを受け付けないと機能しない |
| FR-002 | Must | 本案件の主目的 |
| FR-003 | Must | 実運用でフレーム数不一致は頻繁に発生する |
| FR-004 | Should | FPS自動取得は便利だが、手動指定でも代替可能 |
