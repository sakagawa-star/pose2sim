# 015: Kalmanフィルタの短シーケンスクラッシュ修正 — 要求仕様書

## 1.1 プロジェクト概要

- **何を修正するか**: `Pose2Sim/filtering.py` の `kalman_filter_1d` 関数が、有効データが2〜3フレームしか連続しないシーケンスで `IndexError` を起こす不具合を修正する
- **なぜ修正するか**: 人が一時的にカメラから消える録画データで `Pose2Sim.filtering()` が完了できない
- **誰が使うか**: Pose2Sim利用者（sakagawa）
- **どこで使うか**: `Pose2Sim.filtering()` パイプラインステップ

## 1.2 用語定義

| 用語 | 定義 |
|------|------|
| 有効データ | 非NaN かつ 非ゼロの座標値 |
| シーケンス | 連続する有効データのフレーム区間 |
| `nb_derivatives` | Kalmanフィルタの状態ベクトルに含む微分階数。現在のコードでは `3`（位置・速度・加速度） |

## 1.3 機能要求一覧

### FR-001: Kalmanフィルタの最低シーケンス長チェック

- **機能名**: 短シーケンスのスキップ
- **概要**: `kalman_filter_1d` で、`kalman_filter` が処理可能な最低フレーム数未満のシーケンスをフィルタリング対象から除外する
- **入力**: TRCファイルの各座標列（NaN・ゼロを含む時系列データ）
- **出力**: フィルタリング済みの座標列。短すぎるシーケンスは元の値のまま保持される
- **受け入れ基準**:
  1. `nb_derivatives=3` の場合、有効フレーム数が3以下のシーケンスはフィルタリングせず元の値を保持する
  2. `20260319-dgtw-lab2_0-108003.trc` に対して `Pose2Sim.filtering()` がエラーなく完了する
  3. 十分な長さ（4フレーム以上）のシーケンスに対するフィルタリング結果は修正前と同一である

## 1.4 非機能要求

- パフォーマンス: 処理時間に影響なし（条件チェックの変更のみ）
- 互換性: 修正前に正常完了していたデータに対する出力は変わらない

## 1.5 制約条件

- 変更対象は `Pose2Sim/filtering.py` の428行目のみ
- Pose2Simの他のフィルタ関数（one_euro, gcv_spline, butterworth等）は変更しない
