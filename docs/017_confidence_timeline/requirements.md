# 017: キーポイント信頼度タイムラインCLIツール — 要求仕様書

## 1. プロジェクト概要

- **何を作るのか**: OpenPose形式のJSONファイル群から、指定キーポイントの信頼度をフレーム単位で時系列プロットするCLIツール
- **なぜ作るのか**: 信頼度の時系列変化を可視化し、信頼度に基づいて人物の有無を判定できる閾値が存在するかを確認したい
- **誰が使うのか**: Pose2Sim利用者（データ解析者）
- **どこで使うのか**: Linux環境、Pose2Sim Python環境内

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| 信頼度 | RTMposeが各キーポイントに付与するconfidence score（0.0〜1.0） |
| 人物エントリ | JSONファイルのpeople配列内の1要素。1フレームに複数存在しうる |

## 3. 機能要求一覧

### FR-001: CLI実行

- **概要**: コマンドラインから実行できる
- **入力**:
  - `-j` JSONディレクトリ（必須）
  - `-k` 表示キーポイント名（任意、カンマ区切り。デフォルト: `Nose,Neck,RShoulder,LShoulder`）
  - `-o` 出力ディレクトリ（任意。デフォルト: `{json_dir}_conf_timeline`）
  - `-t` 閾値線（任意、カンマ区切り。例: `0.3,0.5`。未指定時は閾値線なし）
- **出力**: なし（引数のパース結果を後続処理に渡す）
- **受け入れ基準**: `confidence_timeline -j /path/to/json_dir` で実行できる

### FR-002: JSONファイル読み込み

- **概要**: JSONディレクトリ内の全JSONファイルをファイル名順に読み込み、各フレーム・各人物の指定キーポイント信頼度を取得する
- **入力**: JSONディレクトリパス、キーポイント名リスト
- **出力**: list of tuple (frame_idx: int, person_idx: int, keypoint_name: str, confidence: float)
- **受け入れ基準**: 全フレーム・全人物エントリの信頼度を漏れなく取得する

### FR-003: CSV出力

- **概要**: 取得した信頼度データをCSVファイルとして保存する
- **入力**: FR-002の出力データ
- **出力**: CSVファイル（列: `frame, person, keypoint, confidence`）
- **受け入れ基準**: 出力ディレクトリに `confidence_timeline.csv` が生成される

### FR-004: PNGプロット出力

- **概要**: キーポイントごとのサブプロット（縦並び）で信頼度の時系列散布図をPNG保存する
- **入力**: FR-002の出力データ、閾値リスト（任意）
- **出力**: PNGファイル
- **受け入れ基準**:
  - サブプロットはキーポイント数分の段数（デフォルト4段）
  - X軸: フレーム番号、Y軸: 信頼度（0〜1）
  - 各フレーム・各人物エントリの信頼度を点（散布図）としてプロット
  - 人物インデックス（フレーム内の順番）で色分けする。Person 0=青(C0)、Person 1=オレンジ(C1)、Person 2=緑(C2)、Person 3=赤(C3)、Person 4=紫(C4)、Person 5以降=グレー
  - 閾値が指定されている場合、水平破線を描画する
  - 出力ディレクトリに `confidence_timeline.png` が生成される

## 4. 非機能要求

- **パフォーマンス**: 10万フレームを1分以内に処理できる
- **対応環境**: Linux、Python 3.9+、GPU不要
- **依存ライブラリ**: NumPy、matplotlib、tqdm（すべてPose2Sim既存依存）

## 5. 制約条件

- HALPE_26モデル（26キーポイント）前提
- 人物の追跡（フレーム間の同一人物対応付け）は行わない
- 既存CLIツール（`pose_extract_person.py`等）と同じパターンでエントリーポイントを登録する
