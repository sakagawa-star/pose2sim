# 014: 主要人物抽出CLIツール — 要求仕様書

## 1.1 プロジェクト概要

- **何を作るか**: OpenPose形式のJSONディレクトリから、主要人物（患者）のみを抽出して新しいJSONディレクトリに書き出すCLIツール `pose_extract_person`
- **なぜ作るか**: 単一カメラの長時間撮影で、看護師などの一時的な映り込みが含まれるフレームがある。`personAssociation()` は複数カメラ間の対応付けであり、単一カメラ内のフレーム間追跡には使えないため、専用の前処理ツールが必要
- **誰が使うか**: Pose2Simを使うデータ解析者（sakagawa）
- **どこで使うか**: Linux環境、micromamba仮想環境（Pose2Sim）、CLIから実行

## 1.2 用語定義

| 用語 | 定義 |
|------|------|
| OpenPose JSON | 各フレームの2D姿勢推定結果。`people` 配列に各検出人物の `pose_keypoints_2d` を含む |
| `pose_keypoints_2d` | 長さ 78 (= 26キーポイント × 3) の1次元配列。[x0, y0, conf0, x1, y1, conf1, ...] の形式 |
| 有効人物 | conf > 0.1 かつ非NaNのキーポイントが1つ以上ある人物 |
| 主要人物 | 映像の大部分に映っている対象者（患者）。前フレーム追跡で特定する |
| NaN埋め空エントリ | `people` 配列内でキーポイントが全てNaN/conf=0の人物エントリ。実検出ではないため除外対象 |

## 1.3 機能要求一覧

### FR-001: JSONディレクトリ入力

- **機能名**: 入力JSONディレクトリの読み込み
- **概要**: 指定されたディレクトリ内の全JSONファイルをファイル名昇順で読み込む
- **入力**: `-i` オプションで指定するディレクトリパス（1つ）
- **出力**: フレームごとのJSONデータ（内部処理用）
- **受け入れ基準**: 指定ディレクトリ内の全 `*.json` ファイルがファイル名昇順で読み込まれること

### FR-002: 主要人物選択

- **機能名**: フレームごとの主要人物選択
- **概要**: 各フレームの `people` 配列から主要人物1人を選択する
- **入力**: フレームのJSONデータ（`people` 配列）と前フレームの選択結果
- **出力**: 選択された人物1人のキーポイント
- **受け入れ基準**:
  1. 有効人物が0人のフレーム → 空の `people` 配列を出力する
  2. 有効人物が1人のフレーム → その人物をそのまま出力する
  3. 有効人物が2人以上のフレーム → 前フレームの人物に最も近い人物を出力する
  4. 前フレームがない場合（初回 or 直前が0人） → 有効キーポイント数が最多の人物を出力する

### FR-003: JSON出力

- **機能名**: 抽出結果のJSON書き出し
- **概要**: 選択された主要人物のみを含むJSONファイルを出力ディレクトリに書き出す
- **入力**: 選択された人物データ、元のJSONファイル名
- **出力**: 出力ディレクトリに、入力と同名のJSONファイルを作成
- **受け入れ基準**:
  1. 出力JSONの `people` 配列は選択された1人のみを含む（有効人物0人の場合は空配列）
  2. 出力JSONのファイル名は入力JSONと同一
  3. 出力JSONの `pose_keypoints_2d` の内容（座標・信頼度）は入力データと完全一致
  4. 出力JSONのフォーマットは入力と同じ構造（`{"people": [{"pose_keypoints_2d": [...]}]}`）

### FR-004: CLIインターフェース

- **機能名**: コマンドライン引数
- **概要**: argparseで以下の引数を受け付ける
- **入力**:
  - `-i`, `--input`: 入力JSONディレクトリ（必須）
  - `-o`, `--output`: 出力JSONディレクトリ（省略時: `{入力ディレクトリ名}_person` を入力の親ディレクトリに作成）
- **出力**: 処理結果の標準出力ログ
- **受け入れ基準**:
  1. `-i` のみで実行可能
  2. `-o` 省略時、入力が `/path/to/cam01_json` なら出力は `/path/to/cam01_json_person`
  3. 出力ディレクトリが存在しない場合は自動作成する
  4. 出力ディレクトリが既に存在する場合、同名ファイルは上書きされる。既存ファイルの削除は行わない

### FR-005: 処理状況の表示

- **機能名**: 進捗表示とサマリ出力
- **概要**: 処理中にtqdmプログレスバーを表示し、完了時にサマリを標準出力する
- **入力**: 処理対象のフレーム数
- **出力**: プログレスバー、完了時サマリ
- **受け入れ基準**:
  1. tqdmプログレスバーが表示される
  2. 完了時に以下を出力: 総フレーム数、有効人物0人のフレーム数、複数人検出フレーム数、処理時間

## 1.4 非機能要求

- **パフォーマンス**: 97,425フレームを60秒以内に処理完了すること
- **対応環境**: Python 3.9以上、Linux。GPU不要
- **信頼性**: 入力JSONの内容を破壊しない（読み取り専用アクセス）

## 1.5 制約条件

- **使用ライブラリ**: Python標準ライブラリ（json, os, sys, glob, time, argparse）+ numpy + tqdm のみ。追加ライブラリ不可
- **キーポイント形式**: HALPE_26（26キーポイント × 3値）を前提とする
- **信頼度閾値**: 有効キーポイントの判定閾値は `conf > 0.1` 固定
