# 018: オーバーレイ動画 confidence閾値フィルタ — 要求仕様書

## 1. プロジェクト概要

- **何を作るのか**: `pose_overlay_video.py`にconfidence閾値オプションを追加し、指定閾値未満のキーポイント・骨格線を描画しないようにする
- **なぜ作るのか**: 低confidence（推定精度が低い）キーポイントが描画されると、動画が見づらく、キーポイントの挙動確認に支障がある。閾値フィルタにより、信頼性の高いキーポイントのみを可視化して分析精度を上げたい
- **誰が使うのか**: Pose2Sim利用者（データ解析者）
- **どこで使うのか**: Linux環境、Pose2Sim Python環境内

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| confidence | RTMposeが出力するキーポイントごとの推定信頼度。0.0〜1.0の範囲 |
| confidence閾値 | この値未満のキーポイントを描画対象から除外する基準値 |
| キーポイント | 人体の関節位置を示す2D座標と信頼度の組（x, y, confidence） |
| 骨格線 | キーポイント間を結ぶ線。両端のどちらかが非表示ならその骨格線も描画しない |

## 3. 機能要求一覧

### FR-001: confidence閾値CLI引数

- **概要**: `--conf_threshold`（短縮: `-c`）引数でconfidence閾値を指定できる
- **入力**: float値（0.0〜1.0）。デフォルト: 0.0（現行動作と同じ＝全キーポイント描画）
- **出力**: パース済みの閾値をprocess関数に渡す
- **受け入れ基準**: `pose_overlay_video -j /path/to/json_dir -c 0.3` で閾値0.3が適用される

### FR-002: キーポイントのconfidenceフィルタリング

- **概要**: JSON読み込み後、confidence < 閾値のキーポイント座標をNaNに置換する。NaN化されたキーポイントは`draw_keypts()`と`draw_skel()`の既存ロジックにより描画されない
- **入力**: キーポイント配列（26, 3）、confidence閾値（float）
- **出力**: フィルタ済みキーポイント配列（閾値未満のキーポイントの座標がNaN）
- **受け入れ基準**:
  - 閾値0.0: 現行動作と同じ（confidence <= 0のみNaN化）
  - 閾値0.3: confidence < 0.3のキーポイントが描画されない
  - 閾値1.0: 全キーポイントが描画されない（confidence=1.0のキーポイントのみ描画）

### FR-003: 骨格線の連動非表示

- **概要**: 骨格線の両端キーポイントのうち、どちらかがNaN（閾値フィルタまたはconfidence=0で非表示）であれば、その骨格線も描画しない
- **入力**: フィルタ済みキーポイント座標
- **出力**: NaN座標を含む骨格線が描画されない動画
- **受け入れ基準**: `draw_skel()`の既存ロジック（NaN座標のペアをスキップ）により自動的に実現される。追加実装不要

## 4. 非機能要求

- **パフォーマンス**: 閾値フィルタリングによる処理時間の増加は無視できるレベル（NaN比較のみ）
- **後方互換性**: デフォルト値（0.0）で現行動作と完全に同じ出力を生成する
- **対応環境**: Linux、Python 3.9+、GPU不要

## 5. 制約条件

- 既存の`common.py`描画関数（`draw_skel`, `draw_keypts`）は変更しない
- `pose_overlay_video.py`内の変更のみで完結させる
- 閾値フィルタリングはJSON読み込み時に座標をNaN化する方式で実現する（描画関数側の変更不要）

## 6. 優先順位

| ID | 優先度 | 備考 |
|----|--------|------|
| FR-001 | Must | CLI引数がないとフィルタリングを指定できない |
| FR-002 | Must | 本案件の主目的 |
| FR-003 | Must | 既存`draw_skel()`のNaNスキップロジックで自動実現（追加実装不要） |
