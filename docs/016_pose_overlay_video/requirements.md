# 016: 2Dキーポイントオーバーレイ動画生成ツール — 要求仕様書

## 1. プロジェクト概要

- **何を作るのか**: OpenPose形式のJSONファイル群から、フレームごとにキーポイント・骨格を描画した動画を生成するCLIツール。背景は指定した静止画像または黒背景
- **なぜ作るのか**: 別PCでRTMposeを実行した結果（JSONのみ）が送られてくるケースで、元動画がなくても2Dキーポイントの追跡精度・特定フレームの挙動・全体の動きの流れを目視確認したい
- **誰が使うのか**: Pose2Sim利用者（データ解析者）
- **どこで使うのか**: Linux環境、Pose2Sim Python環境内

## 2. 用語定義

| 用語 | 定義 |
|------|------|
| 背景画像 | 動画の全フレーム共通の背景として使う静止画像（元動画から抽出した1フレーム等） |
| キーポイント | 人体の関節位置を示す2D座標と信頼度の組（x, y, confidence） |
| 骨格線 | キーポイント間を結ぶ線（`skeletons.py`のモデル定義に基づく接続） |
| オーバーレイ | 背景画像の上にキーポイント・骨格線を重ねて描画すること |
| 生JSON | RTMposeが出力したOpenPose形式のJSONファイル。複数人物が含まれる場合がある |

## 3. 機能要求一覧

### FR-001: CLI実行

- **概要**: コマンドラインから実行できる
- **入力**:
  - `-j` JSONディレクトリ（必須）
  - `-b` 背景画像パス（任意。未指定時は黒背景）
  - `-o` 出力動画パス（任意。未指定時は `{json_dir}_overlay.mp4`）
  - `--fps` フレームレート（任意。未指定時は30）
  - `--size` 画像サイズ（任意。`WxH` 形式、例: `1920x1080`。背景画像指定時は無視される）
- **出力**: なし（引数のパース結果を後続処理に渡す）
- **受け入れ基準**: `pose_overlay_video -j /path/to/json_dir -b /path/to/image.jpg` で実行できる

### FR-002: JSONファイル読み込み

- **概要**: JSONディレクトリ内の全JSONファイルをファイル名順に読み込み、各フレームの全人物のキーポイントを取得する
- **入力**: JSONディレクトリパス
- **出力**: フレームごとの人物リスト（各人物: X座標(26,)、Y座標(26,)、信頼度(26,)の3配列）
- **受け入れ基準**: OpenPose形式の生JSONを正しくパースし、複数人物も全員描画する

### FR-003: 背景画像の準備

- **概要**: 指定された背景画像を読み込む。背景画像が指定されない場合は黒背景を使用する
- **入力**: 背景画像パス（任意）、画像サイズ（任意、`--size` 引数）
- **出力**: BGR画像（numpy.ndarray, shape=(H,W,3), dtype=uint8）
- **受け入れ基準**:
  - 背景画像指定あり: その画像を読み込む。`--size` は無視
  - 背景画像指定なし + `--size` あり: 指定サイズの黒背景画像を生成する
  - 背景画像指定なし + `--size` なし: デフォルト1920x1080の黒背景画像を生成する

### FR-004: キーポイント・骨格の描画

- **概要**: 各フレームについて、背景画像のコピー上にキーポイント（信頼度に応じた色付き丸）と骨格線（左右・中央で色分け）を描画する。複数人物が検出されている場合は全員を描画する
- **入力**: 背景画像、フレームのキーポイント・信頼度
- **出力**: 描画済み画像
- **受け入れ基準**: `common.py`の`draw_keypts()`と`draw_skel()`を使用し、`poseEstimation.py`の描画処理と同等の見た目になる

### FR-005: 動画書き出し

- **概要**: 描画済みフレーム画像を連結してMP4動画ファイルとして出力する
- **入力**: 描画済みフレーム画像のシーケンス、FPS、出力パス
- **出力**: MP4動画ファイル
- **受け入れ基準**: OpenCVのVideoWriterでmp4vコーデックの動画が生成される

### FR-006: フレーム番号表示

- **概要**: 各フレームの左上にフレーム番号を表示する
- **入力**: フレームインデックス
- **出力**: フレーム番号が描画された画像
- **受け入れ基準**: 白文字で「Frame: {番号}」が左上に表示される

## 4. 非機能要求

- **パフォーマンス**: 10万フレームを5分以内に処理できる（描画がボトルネックにならない）
- **対応環境**: Linux、Python 3.9+、GPU不要
- **依存ライブラリ**: OpenCV、NumPy、matplotlib、tqdm（すべてPose2Sim既存依存）

## 5. 制約条件

- `Pose2Sim/common.py`の既存描画関数（`draw_skel`, `draw_keypts`）を使用する
- `Pose2Sim/skeletons.py`の`HALPE_26`モデルを骨格定義として使用する
- 既存CLIツール（`pose_confidence_analyze.py`等）と同じパターンでエントリーポイントを登録する
- 入力JSONは生のRTMpose出力（`pose_extract_person` 等で前処理されていない）を想定する
