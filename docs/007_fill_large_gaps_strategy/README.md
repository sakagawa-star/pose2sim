# 007: fill_large_gaps_with 戦略の検討 [クローズ — nan採用]

## 概要

三角測量後の大きなギャップ（`interp_if_gap_smaller_than`=15フレーム超）の充填方法を評価。
3戦略（`last_value`, `nan`, `zeros`）を比較し、`nan`が最良と結論。

## 対象データ

`20260227-dgtw2-lab2` (4台カメラ, 1920x1080, 1770フレーム, 26マーカー)

## コード分析

### ギャップ処理フロー（triangulation.py:888-926）

1. **Phase 1（行892）**: gaps ≤ 15フレーム → `interpolate_zeros_nans()`で線形補間
2. **Phase 2（行922-926）**: 残りのギャップ → `fill_large_gaps_with`で充填

### 3つの選択肢の実装

| 戦略 | 実装 | フィルタリング挙動 | LSTM互換性 |
|------|------|-------------------|-----------|
| `last_value` | `ffill().bfill()` → 残NaN/infを0に | 定値区間を有効データとして処理 | 動作する |
| `zeros` | NaN/infを0に置換 | `eq(0)`マスクで欠損扱い→nanと同等 | 原点座標→無意味 |
| `nan` | 何もしない | NaNで分割→各セグメント独立処理 | **NaN非対応** |

### 重要発見

- filtering.py行461: `mask = np.isnan(col) | col.eq(0)` → **zerosとnanはフィルタで同じ挙動**
- よって `zeros` は `nan` の下位互換（フィルタは同じ、LSTMに無意味な値を入力）
- **実質的な比較は `last_value` vs `nan` の2択**

## LSTM MarkerAugmenterへの影響

- `nan`戦略ではLSTMがNaNフレームに対してNaN出力を生成する
- ただし`last_value`での凍結座標もLSTMにとって意味のある入力ではない
- **どちらの戦略でもギャップ部分のLSTM出力は信頼できない**
- `nan`は「欠損」を正直に表現するため、後段解析で誤解を防げる

## 結論

### 推奨: `fill_large_gaps_with = "nan"`

**理由**:
1. **3/4指標で改善**（BoneCV -0.9%, Smooth -1.2%, L-R -0.6%）
2. `last_value`は最大570mmのテレポートジャンプを生む → 物理的に不可能な軌跡
3. NaN率0.5%増加は許容範囲（5マーカー×限定的なフレーム範囲のみ）
4. NaNは「データなし」を正直に表現 → 後段解析（OpenSim等）での誤解を防止

### 適用方法

Config.tomlで以下を変更:
```toml
[triangulation]
fill_large_gaps_with = "nan"    # 変更: "last_value" → "nan"
```

### 注意事項

- `zeros`は`nan`の下位互換のため不採用（フィルタ挙動同一、LSTM入力が無意味）
- LSTM MarkerAugmenterはNaNフレームでNaN出力を生成するが、凍結座標での出力も信頼できないため実質的なデメリットなし
- `interp_if_gap_smaller_than = 15`（15フレーム以下のギャップは線形補間）は変更不要
