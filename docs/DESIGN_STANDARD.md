# docs/DESIGN_STANDARD.md

## 機能設計書 記述基準
本基準に従って機能設計書を作成すること。

---

### 1. 文書構成（必須セクション）

#### 1.1 対応要求マッピング
- 本設計書がカバーする要求仕様書のIDを列挙する
- 要求IDと設計セクションの対応表を冒頭に置く

#### 1.2 システム構成
- モジュール構成図（どのファイル/クラスが何を担当するか）
- モジュール間の依存関係
- ディレクトリ構成（実際のパス）

#### 1.3 技術スタック
- 言語・バージョン（例: Python 3.11）
- 使用ライブラリとバージョン（例: OpenCV 4.9.0, PyQt6 6.6）
- パッケージ管理方法（例: micromamba, requirements.txt）
- ライブラリの選定理由（簡潔に）

#### 1.4 各機能の詳細設計
機能ごとに以下を記述する:

##### データフロー
- 入力データの型・形式・単位・値域
  - 例: 画像 → numpy.ndarray, shape=(H,W,3), dtype=uint8, BGR色空間
  - 例: 設定値 → float, 0.0〜1.0, デフォルト0.5
- 出力データの型・形式・単位・値域
- データ変換の過程（中間データも含む）

##### 処理ロジック
- アルゴリズムの手順を擬似コードまたはステップで記述
- 条件分岐はすべての分岐先を明記する
- ループがある場合、終了条件を明記する

##### エラーハンドリング
- 想定されるエラーを列挙する
- 各エラーに対する処理を具体的に定義する:
  - 検出方法
  - リカバリ動作（リトライ回数、フォールバック、ユーザー通知等）
  - ログ出力内容

##### 境界条件
- 入力が空/ゼロ/1件/上限値の場合の振る舞い
- リソース不足（メモリ、ディスク、カメラ台数不足）時の振る舞い

#### 1.5 状態遷移（GUI/ステートフル処理がある場合）
- 状態一覧（例: idle, recording, processing, error）
- 状態遷移表またはMermaid状態遷移図
- 各遷移のトリガー条件
- 各状態でのUI要素の有効/無効状態
- 不正な遷移が要求された場合の振る舞い

#### 1.6 ファイル・ディレクトリ設計
- 入出力ファイルのパス規約
- ファイル命名規則（タイムスタンプ形式等）
- 設定ファイルの形式（TOML/YAML/JSON）とスキーマ
- 設定項目ごとのデフォルト値・型・値域

#### 1.7 インターフェース定義
- 公開関数/メソッドのシグネチャ
  - 関数名、引数名、引数の型、戻り値の型
  - 各引数のデフォルト値
- クラスの責務（1クラス1責務を原則とする）
- モジュール間の呼び出し方向（循環依存の禁止）

#### 1.8 ログ・デバッグ設計
- ログレベルの使い分け基準（DEBUG/INFO/WARNING/ERROR）
- 各処理のログ出力ポイント
- ログフォーマット

---

### 2. 記述ルール

#### 2.1 禁止表現（要求仕様書と同一）
曖昧表現を使わない。すべて具体値・具体条件で記述する。

#### 2.2 実装判断ゼロの原則
設計書を読んだClaude Codeが「自分で判断する」場面をゼロにする。
以下のチェックで確認する:
- 「この設計を読んで、2通り以上の実装が考えられる箇所はないか？」
- もしあれば、設計が不十分なので詳細化する

#### 2.3 コード例の活用
複雑なロジックや、ライブラリの使い方に癖がある場合は
短いコードスニペットを設計書内に含めてよい。
ただし設計書のコード例は「意図の伝達」が目的であり、
そのままコピーして使うものではないことを明記する。

#### 2.4 設計判断の記録
複数の選択肢があり得た設計判断については:
- 採用案
- 却下案とその理由
を簡潔に記録する（ADR: Architecture Decision Recordの簡易版）
これにより、Claude Codeが別の方法に変えてしまうことを防ぐ。
```

## 3つのファイルの関係
```
CLAUDE.md（1行ずつ参照を書くだけ）
  ├─ 要求仕様書作成時 → docs/REQUIREMENTS_STANDARD.md に従え
  ├─ 機能設計書作成時 → docs/DESIGN_STANDARD.md に従え
  └─ レビュー実行時   → docs/REVIEW_CRITERIA.md に従え

.claude/commands/review.md（Slashコマンド）
  └─ /review で3つの基準すべてを踏まえてレビュー実行
