# 010: 2Dキーポイント信頼度分析 — 要求仕様書

## 1. ツール概要

| 項目 | 内容 |
|------|------|
| ツール名 | `pose_confidence_analyze` |
| 目的 | 2Dポーズ推定の信頼度を多角的に分析し、三角測量品質への影響を評価する |
| 入力 | poseディレクトリパス（`cam*_json`を含むディレクトリ） |
| 出力 | コンソールレポート + CSV + ヒートマップ画像 |
| 設計パターン | 既存の`trc_evaluate.py`と同じCLI構成に準拠 |

## 2. 機能要件

### FR-1: データ読み込み
- poseディレクトリ内の`cam*_json/`サブディレクトリを自動検出する
- 各カメラの全フレームJSONを読み込み、信頼度値を抽出する
- 26キーポイント × 全フレームの信頼度行列を各カメラについて構築する
- キーポイント名はHALPE_26の定義に従う

### FR-2: 分析1 — カメラ別・キーポイント別の信頼度統計
- カメラ×キーポイントの行列で以下の統計量を算出する:
  - 平均（mean）
  - 中央値（median）
  - 標準偏差（std）
  - 最小値（min）
  - パーセンタイル: 5th, 25th, 75th, 95th
- 閾値（デフォルト0.4）未満のフレーム割合を算出する
- 各キーポイントの全カメラ平均も算出する（キーポイント固有の傾向把握のため）

### FR-3: 分析2 — 信頼度帯域分布
- 以下の帯域に分類し、カメラ別・キーポイント別のフレーム数と割合を算出する:
  - `[0, 0.4)`: 低信頼（現閾値で除外される）
  - `[0.4, 0.6)`: 危険ゾーン（閾値は超えるが低品質の可能性）
  - `[0.6, 0.8)`: 中信頼
  - `[0.8, 1.0]`: 高信頼
  - `(1.0, ∞)`: 超高信頼（1.0超、参考）
- 「危険ゾーン」（0.4-0.6）に該当するデータの総量を明示する
- 閾値を0.5/0.6に引き上げた場合の追加除外率をシミュレーションする

### FR-4: コンソールレポート出力
- 分析1のサマリーテーブル（キーポイント×カメラの平均信頼度）を表示する
- 分析2の帯域分布サマリーを表示する
- 問題キーポイント（低信頼度の傾向があるもの）をハイライトする
- 閾値引き上げシミュレーション結果を表示する

### FR-5: CSV出力
- 分析1の全統計量をCSVファイルに保存する（1行＝1カメラ×1キーポイント）
- 分析2の帯域分布をCSVファイルに保存する
- デフォルトファイル名: 入力ディレクトリ名に基づいて自動命名する

### FR-6: ヒートマップ画像出力
- カメラ×キーポイントの平均信頼度ヒートマップを生成・保存する
- カメラ×キーポイントの危険ゾーン（0.4-0.6）割合ヒートマップを生成・保存する
- matplotlib使用、保存先はCSVと同じディレクトリとする

## 3. 非機能要件

### NFR-1: CLI仕様
```
pose_confidence_analyze -p <pose_dir> [-t <threshold>] [-o <output_dir>] [--no-plot]
```

| 引数 | 必須 | デフォルト | 説明 |
|------|------|-----------|------|
| `-p/--pose-dir` | Yes | — | poseディレクトリパス |
| `-t/--threshold` | No | `0.4` | 現行の信頼度閾値 |
| `-o/--output` | No | `<pose_dir>/confidence_analysis/` | 出力ディレクトリ |
| `--no-plot` | No | False | ヒートマップ画像を生成しない |

### NFR-2: 依存ライブラリ
- numpy, matplotlib のみ（Pose2Simの既存依存に含まれる）
- pandas は使用しない（JSONから直接numpy配列へ）

### NFR-3: 実行時間
- 全データ（~7,200フレーム × 4カメラ）の処理が30秒以内に完了すること

### NFR-4: 配置
- ファイル: `Pose2Sim/Utilities/pose_confidence_analyze.py`
- エントリーポイント: `pyproject.toml`の`[project.scripts]`に追加
