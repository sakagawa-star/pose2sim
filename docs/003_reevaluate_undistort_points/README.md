# 003: undistort_points の再評価 [完了]

**依存**: 002（intrinsicsキャリブレーション改善）の完了が前提

## 背景

`undistort_points = true` にすると再投影誤差が増加（9.9px→11.4px）、カメラ除外率が倍増（0.48→0.96台）。cam01/cam03の除外率が35-38%に達する。

**原因**: cam01/cam03の望遠レンズの歪みモデル精度が不十分。

## 002完了後の再評価結果

002案A（k3パラメータ解放）によるキャリブレーション改善後、undistort_points=true/falseの両方でTRC品質を比較。

**結果**: 再投影誤差・カメラ除外率は改善しなかったが、**TRC品質は全指標で改善**:

| 指標 | undistort=false (参考) | undistort=true (k3解放後) |
|------|----------------------|--------------------------|
| Bone CV | 18.0% | 16.4% |
| Smoothness | 0.0189 | 0.0139 (-26%) |
| L-R Diff | 6.3% | 4.9% |

## 結論

**`undistort_points = true` を推奨**。カメラ除外が増えても、残ったカメラでの三角測量の質が向上するため、最終的なTRC品質は改善する。

002の案B/C結果も参照:
- 案B（alpha=0）: cam除外率改善だがSmooth/L-R悪化で不採用
- 案C（画像品質フィルタ）: cam01のみ13枚除外で効果限定的、不採用
- 案A設定を維持。コードは`calibration.py`に残置（害なし）
